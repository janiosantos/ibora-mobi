Nosso objetivo inicial é criar backend para a  um app estilo Uber/99 . Na raiz do  projeto tem pasta "repo" com diversos projetos do github com funcionalidades que vamos precisar backend que irão servir de inspiração. O o workspace de trabalho do backend será na pasta raiz backend do repositorio. La você vai  encontrar o prompt para instruçoes globais que vao direcionar os trabalhos. Todo inicio de tarefa será precedido de um arquivo claro com instruções  do já foi realiazado, ultima descrição da ultima tarefa/etapa concluida com sucesso, e o que será implementado naquela proxima tarefa/etapa. Esse arquivo deverá ser atualizado e salvo em todo inicio de tarefa com o respectivo commit para o repositorio remoto. E atualizado no final da tarefa e realizado o respectivo commit para repositorio remoto ao final de cada tarefa/etapa concluído. Tal procedimento visa evitar erros de continuidade como o sistema atigir limites de uso. O nome sujestivo para arquivo PROGRESSO.md , caso o arquivo fique muito grande poderá ser gerado um resumo para  proximo arquivo por exemplo PROGRESSO-CURRENT.md e o arquivo anterior nomeado para PROGRESSO-01.md, assim que PROGRESSO-CURRENT.md ficar muito grande gere resumo novamente para PROGRESSO-CURRENT.md e renomeie o arquivo anterior para PROGRESSO-02.md, assim sucessivamente, apenas sugestão, voce pode controlar de outra maneira mais eficiente, contato que mantenha registro para que consigamos da continuiedade em outra sessão.
